# 保存方式とPDFチェックの挙動

この資料は、kojiPDF の「保存方式」と「PDFを詳しくチェック」の挙動を整理した開発者向けメモです。
README やアプリ内ヘルプへ説明を転記するときの原稿としても使えます。

## 保存方式（ドロップダウン）

保存方式では、PDF作成時の速度とファイルサイズのバランスを選びます。

### ユーザー向け解説

#### 速度重視

保存時間を短くする設定です。ファイルサイズは大きくなりやすいですが、大量ページや急ぎの作成に向いています。

#### バランス

通常の推奨設定です。保存時間とファイルサイズのバランスを取ります。

#### 圧縮重視

ファイルサイズを小さくすることを優先します。保存に時間がかかる場合があります。

また、事前チェックで修復可能なPDF warning が見つかった場合、結合前に処理対象フォルダ内のPDFを修復することがあります。修復前のPDFは、出力PDFと同じ場所の `error_files` フォルダに退避されます。

Office変換を使う場合は、コピー後の一時フォルダ内PDFが処理対象になるため、通常は元フォルダのPDF自体は変更されません。Office変換を使わない場合は、選択した元フォルダ内のPDFが処理対象になります。

### デベロッパー向け解説

#### 速度重視

`garbage=1`, `deflate=False` で保存する。

圧縮処理を抑え、保存速度を優先する。

#### バランス

`garbage=1`, `deflate=True` で保存する。

不要オブジェクト整理は軽めにしつつ、ストリーム圧縮を行う。通常のデフォルト設定。

#### 圧縮重視

`garbage=4`, `deflate=True` で保存する。

ファイルサイズ削減を優先するため、最終保存時の処理負荷は高くなる。

圧縮重視では `is_smaller_file_save_mode()` が真になり、事前チェックで修復可能な MuPDF warning が検出された場合、結合前に `repair_problem_source_pdfs()` による修復を試す。

修復できたPDFは処理対象フォルダ内で修復済みPDFに置き換え、修復前PDFは出力PDFと同じフォルダ配下の `error_files` に退避する。

修復に失敗した warning は記録し、元のPDFのまま処理を続行する。

`error` 扱いのPDF、たとえば空ファイル、暗号化PDF、開けないPDFなどは自動修復せず、処理を停止する。

## PDFを詳しくチェック（トグル）

このトグルは、PDF結合に失敗する場合、壊れたPDFが混ざっていそうな場合、提出前に念入りに確認したい場合に使う。

通常はOFFのままで問題ない。

### ユーザー向け解説

次のような場合にONにしてください。

- PDF結合に失敗する場合
- 壊れたPDFが混ざっていそうな場合
- 提出前に念入りに確認したい場合
- 前回のPDF作成で警告やエラーが出た場合

#### OFFの場合

- 高速チェックだけ実行します
- 問題がなければそのままPDF作成を進めます
- 修復可能な warning は、圧縮重視の場合のみ自動処理します
- なるべく確認画面を出さずに処理します

#### ONの場合

- 各PDFをより詳しくチェックします
- 問題や修復結果があれば画面で確認できます
- 保存時の warning も確認対象になります
- 壊れたPDFの確認、提出前確認、前回失敗時の再実行に向いています

### デベロッパー向け解説

このトグルは、従来の `preflight_detail_repair` と `preflight_confirm` をユーザー向けに1つへまとめる想定。

ONの場合:

- `preflight_detail_repair=True`
- `preflight_confirm=True`

OFFの場合:

- `preflight_detail_repair=False`
- `preflight_confirm=False`

OFFでは `run_fast_preflight_check()` を実行する。

高速チェックではPDFを開き、先頭ページを読み込めるかを確認する。処理時間を抑えるため、全ページの詳細検査は行わない。

ONでは `run_detail_preflight_repair()` を実行する。

詳細チェックでは、しおり、全ページ読み込み、テキスト取得、リンク、注釈、ページ挿入を確認する。結合時に問題になりやすいPDFを事前に検出しやすい。

`preflight_confirm=True` の場合、チェック結果、修復結果、保存時 warning をユーザーに表示し、必要に応じて続行、修復、無視、終了を選ばせる。

圧縮重視の場合のみ `repair_source_warnings=True` となる。

この場合、修復可能な warning があるPDFに対して、結合前の修復を試す。修復できた場合は処理対象フォルダ内のPDFを置き換え、修復前PDFを `error_files` に退避する。詳細チェックON時に修復が成功した場合は、修復後PDFで結合処理をやり直す。

## `error_files` フォルダの場所

`error_files` フォルダは、出力PDFを保存する場所と同じフォルダに作られる。

例:

- 出力PDF: `C:\work\成果品\merged.pdf`
- 退避先: `C:\work\成果品\error_files\`

Office変換ONで一時フォルダ内PDFを修復した場合でも、修復前PDFの退避先は一時フォルダ内ではなく、出力PDFの隣の `error_files` になる。
